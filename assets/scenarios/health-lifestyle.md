# 场景预设：健康 / 生活方式决策

## 触发关键词
"要不要做这个手术" / "换医院" / "改饮食方案" / "戒烟戒酒" / "搬家换城市" / "继续健身/停掉"

## 推荐角色配置（5 角色 standard mode）
1. 数据派（医学/科学证据等级）
2. Bayesian 更新派（成功率 / 副作用概率）
3. 怀疑派（替代方案被忽略的可能）
4. 用户派（生活质量影响 + 心理负担）
5. Steelman 派（多种治疗思路的最强版本）

## 推荐 mode
`standard`（3 轮 + 红队）；如涉及手术/重大医疗决策强制 `deep`

## 议题重述格式
`【议题】在 [健康问题] 上面对 [选项A/B/C]，需要平衡 [收益与风险]`
`【类型】判断型 / 选择型`
`【成本】极高（不可逆 + 健康层面）`

## required_inputs（v0.3.0 强制门禁）

```yaml
required_inputs:
  - 当前生理指标：相关检查/检验数值（具体数字 + 参考区间）
  - 医生意见：是否已咨询医师 + 给出的方案 + 医师专业领域
  - 干预选项：≥ 2 个具体方案（不接受"试试看"这种含糊表述）
  - 时长承诺：用户愿意/可以坚持的时长
  - 个人目标：治愈 / 控制 / 缓解 / 生活质量优先（明确优先级）
```

## 输入预处理（兼容旧版口径）
让用户给出：
- 当前健康状况 + 已尝试方案
- 医生建议 + 替代方案
- 个人目标（治愈 / 控制 / 缓解 / 生活质量）
- 风险承受度

## 输出额外要求
- **证据等级标注**：每条医学论据标注证据强度（RCT / observational / case study / 经验之谈）
- **副作用矩阵**：选项 × 副作用类型 × 概率
- **生活质量影响时间线**：短期（1月）/ 中期（半年）/ 长期（5年）

## ⚠️ 强制提示（顶部，不可省略）
> 本辩论**不构成医疗建议**。所有结论必须与持证医师讨论后采纳。涉及紧急症状/生命危险时立即就医，不要等本辩论。

## 已知陷阱
- **过度自信单一医生**：第二医疗意见的价值常被低估
- **过分依赖网络信息**：医学决策需要医师专业评估
- **现状偏差**：人倾向不改变现有方案——但有时改变才是对的
- **回避不适**：不愿正视严重诊断而拖延决策
