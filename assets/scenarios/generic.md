# 场景预设：通用决策（兜底）

## 触发关键词
当议题不匹配任何具体场景时使用。或当用户主动说"通用模式"。

## 推荐角色配置（5 角色 standard mode）
1. 数据派
2. 怀疑派
3. Steelman 派
4. Bayesian 更新派
5. 第一性原理派

> 不预设领域专家角色（如投资人/用户派/执行派）——根据议题在第一轮后动态判断是否需要补充。

## 推荐 mode
`standard`（3 轮 + 红队）

## 议题重述格式
`【议题】[原议题]`
`【类型】[判断型/选择型/审查型/探索型]`
`【成本】[低/中/高]`

## required_inputs（v0.3.0 强制门禁 — 兜底场景，最低门槛）

```yaml
required_inputs:
  - 议题：精确陈述（不接受"我最近在想 X"这种模糊表述）
  - 决策窗口：什么时候必须做出决定
  - 不可逆程度：自评 低 / 中 / 高
  - 已知约束：用户已声明的硬约束（钱 / 时间 / 能力 / 人 / 法 / 伦理）
```

兜底场景的门禁是最低底线——若连议题精确陈述都做不到，主持人**反推用户先把议题压精确**。

## 输入预处理（兼容旧版口径）
让用户给出：
- 议题背景
- 已有信息 / 已知约束
- 决策时间窗口
- 决策不可逆性自评

## 第一轮后的动态调整
主持人根据第一轮发言判断**是否需要补充领域专家角色**：
- 涉及钱 → 补充 投资人
- 涉及人 → 补充 用户派
- 涉及落地 → 补充 执行派
- 涉及未来 → 补充 战略派

如需补充，在第二轮加入新角色（带"中途加入"标签，参考前 1 轮记录后独立发言）。

## 输出
按 `assets/templates/verdict.md` 标准格式输出。

## 注意
通用模式可能不如专门场景精准。如用户议题明显匹配某个具体场景，主持人应主动建议切换。
