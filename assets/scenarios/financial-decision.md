# 场景预设：财务 / 投资决策

## 触发关键词
"要不要买房" / "投资这只股票" / "理财方案" / "资产配置" / "要不要贷款" / "要不要创业拿融资"

## 推荐角色配置（5 角色 standard mode）
1. Bayesian 更新派（投资类 base rate + 凯利公式仓位估算）
2. 投资人 / 财务视角（ROI / 现金流 / 流动性）
3. 数据派（具体标的的关键指标）
4. 怀疑派（黑天鹅情景）
5. Steelman 派（多空双方最强版本）

## 推荐 mode
`standard`（3 轮 + 红队）；如金额 > 用户净资产 50% 强制 `deep`

## 议题重述格式
`【议题】[金额] 投入 [标的]，预期持有 [时长]。是否值得？仓位是否合理？`
`【类型】判断型`
`【成本】视金额 — 必须强制风险等级评估`

## required_inputs（v0.3.0 强制门禁）

```yaml
required_inputs:
  - 投资标的：具体名称（股票代码 / 房产具体位置 / 项目名 / 基金代码）
  - 本金占比：金额 + 占总净资产比例
  - 时间框：预期持有时长 + 资金锁定期 + 退出场景
  - 替代方案对照：这笔钱不投这个的最优替代用途及其预期回报基线
  - 风险承受度：保守 / 中性 / 激进 + 最坏情况可承受损失上限
```

## 输入预处理（兼容旧版口径）
让用户给出：
- 总净资产 + 流动性资产
- 标的具体描述（股票代码 / 房产位置 / 项目名）
- 持有时长 + 退出场景
- 风险承受度自评（保守 / 中性 / 激进）

## 输出额外要求
- **风险敞口表**：最坏情况下损失多少 / 占净资产几成
- **凯利公式仓位建议**（如适用）
- **流动性评估**：什么时候能卖、卖出折扣
- **替代用途对照**：这笔钱不投这个，预期回报基线

## ⚠️ 财务决策免责声明（强制输出）
在裁决卡顶部固定挂：
> **本辩论不构成投资建议**。所有 base rate 和概率估算来自公开数据/历史案例，不保证未来。重大财务决策建议同时咨询持牌财务顾问。

## 已知陷阱
- **过度自信偏差**：人对自己的判断高估约 30%
- **回报-风险错配**：用户经常只看回报忽略风险方差
- **杠杆诱惑**：贷款投资时杠杆放大双向，必须显式列出
- **市场时机谬误**：试图择时几乎总是低效的
