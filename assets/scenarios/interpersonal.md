# 场景预设：人际 / 关系决策

## 触发关键词
"要不要分手" / "要不要结婚" / "要不要生孩子" / "要不要和这个人合伙" / "和家人闹矛盾"

## 推荐角色配置（4-5 角色 standard mode；temperament 整体调温和）
1. 用户派（共情 + 场景还原；这里"用户"扩展为"所有相关方"）
2. Bayesian 更新派（同类决策结果的统计分布）
3. Steelman 派（构造对方立场的最强版本）
4. 第一性原理派（剥离社会期待看底层需求）
5. （可选）怀疑派（标注风险但不强势）

## 推荐 mode
`standard`，但**所有角色 prompt 加柔性约束**：避免冷血结论 / 不简化为利益计算 / 承认情感本身有内在价值

## 议题重述格式
`【议题】在 [关系/对象] 上面对 [具体问题]，倾向 [当前判断]，需要平衡 [关键张力]`
`【类型】判断型，含情感与价值取舍`
`【成本】极高（不可逆 + 跨人生影响）`

## required_inputs（v0.3.0 强制门禁）

```yaml
required_inputs:
  - 关系性质：亲属 / 伴侣 / 朋友 / 同事 / 合伙人 + 关系时长
  - 关键事件时间线：已发生的具体事件 + 时间序列（≥ 3 个事件，便于 base rate）
  - 双方诉求：用户的核心需求 + 对方的核心需求（双方都要列，否则只是一面之词）
  - 决策时间窗口：紧迫 / 不紧迫 + 是否有外部 deadline
```

## 输入预处理（兼容旧版口径）
让用户给出：
- 关系背景 + 时长 + 关键事件
- 双方诉求差异
- 用户自己的核心需求（不能省略 — 很多人决定时已经忘了自己要什么）
- 决策时间窗口（紧迫 / 不紧迫）

## 输出额外要求
- **多方利益矩阵**：用户 / 对方 / 其他相关人
- **底层需求识别**：剥离表面诉求看真正的需求层次（Maslow 框架）
- **沟通建议**：辩论裁决之外，给出"如果决定 X，怎么沟通"的脚本

## ⚠️ 强制提示（顶部）
> 这是一个涉及人际/情感的决策。AI 辩论可以帮你梳理思路，但**最终判断必须由你自己做**。如涉及严重心理困扰或安全问题（家暴/抑郁/自伤），请优先寻求专业咨询师或心理援助热线。

## 已知陷阱
- **过度理性化**：把情感决策当 ROI 计算会失真
- **忽略对方主体性**：辩论时容易把对方变成被分析的客体
- **当下情绪放大**：用户在情绪中决策时容易过度悲观或乐观
- **价值观替代**：AI 不应试图替代用户的价值观，只提供视角
