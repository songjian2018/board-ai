# 场景预设：多方案取舍

## 触发关键词
"A 和 B 怎么选" / "几条路径选哪个" / "几个项目优先级" / "时间该花在哪"

## 推荐角色配置（5 角色 standard mode）
1. 战略派（长周期视角 / 终局推演）
2. 投资人（ROI / 机会成本 / 风险调整后回报）
3. 执行派（每个方案的落地难度差异）
4. 怀疑派（每个方案各自的失败路径）
5. 未羊视角（决策启发式 + 跨项目耦合）

## 推荐 mode
`standard`（3 轮 + 红队）

## 议题重述格式
`【议题】在 [选项A] / [选项B] / [选项C] 中选择哪个？是否可以并行？`
`【类型】选择型`
`【成本】中-高（取决于选项规模）`

## required_inputs（v0.3.0 强制门禁）

```yaml
required_inputs:
  - 候选选项：≥ 2 个（A / B / C，每个有名字+一句话描述）
  - 每个选项的 5 维属性：成本 / 收益 / 风险 / 时间窗口 / 不可逆性（必须 5 维齐全，缺则 abstain）
  - 资源池约束：可用时间 / 资金 / 精力 / 认知带宽 上限
  - 互斥/并行关系：选项之间是互斥/可并行/可串行
```

## 输入预处理（兼容旧版口径）
- 让用户给每个选项提供：愿景 / 投入估算 / 时间窗口 / 已有资源
- 主持人识别选项之间是 **互斥** / **可并行** / **可串行**
- 若选项间存在显著资源争夺，必须列出"资源池"（时间/钱/精力/认知带宽）

## 输出额外要求
- **方案对比矩阵**：
  ```
  | 维度 | 方案 A | 方案 B | 方案 C |
  |---|---|---|---|
  | 投入 |  |  |  |
  | 5 年终局 |  |  |  |
  | 机会成本 |  |  |  |
  | 失败概率 |  |  |  |
  | 学习/认知收益 |  |  |  |
  | 与其他项目耦合度 |  |  |  |
  ```
- **决策树**：什么前提下选 A，什么前提下选 B，什么前提下并行
- **节奏建议**：是否分阶段、串行、并行的具体调度

## 已知陷阱
- **逃避决策**：用户经常想"全要"，但资源池有限；主持人必须强迫面对取舍
- **沉没成本**：用户偏向已经投入过的方案——必须显式标注"已投入"不计入决策
- **新鲜感偏好**：用户偏向最新看到的方案——必须强迫各方案获得平等评估

## 与其他 skill 联动
- 决策完成后选定一条路径，串联 `apex-protocol` 进入立项规划
- 若决策"先验证再决定"，串联本 skill 的 `项目立项决策` 场景为每个候选选项做单独的立项辩论
