# 场景预设：项目立项决策

## 触发关键词
"我该不该做这个项目" / "这个想法值不值得做" / "立项决策" / "要不要 all in"

## 推荐角色配置（7 角色 deep mode）
1. 战略派（5 年终局 + 护城河来源）
2. 投资人（ROI / 机会成本 / 止损纪律）
3. 用户派（PMF 风险 / 采纳门槛）
4. 数据派（市场规模 / 单位经济模型）
5. 执行派（落地路径 / 工程债 / 资源现实）
6. 怀疑派（每轮魔鬼代言）
7. 未羊视角（既有项目时间冲突 + 跨项目协同）
+ 红队（最后一轮压测）

## 推荐 mode
`deep`（4 轮 + 双红队 + 二次裁决）—— 立项决策不可逆性高，值得多花时间

## 议题重述格式
`【议题】是否在 [时间] 启动 [项目]？投入是 [时间预算]/[现金预算]？`
`【类型】判断型`
`【成本】高（不可逆 + 重大资源投入）`

## required_inputs（v0.3.0 强制门禁 — 缺一即拒启动辩论）

```yaml
required_inputs:
  - 项目立项书 / 一页纸愿景（目标客户 + 价值主张 + 商业模式假设）
  - 预算上限：人月数 / 现金预算 / 储蓄跑道
  - 团队配置：核心成员数 + 角色（产品/工程/市场/运营）
  - 不可逆点清单：哪些决策做完反悔代价大（雇人 / 注册公司 / 签长约 / 开发架构选型）
  - 替代选项：如果不做这个项目，时间和钱会做什么（机会成本对照）
  - 时间窗口约束：市场窗口 / 资金窗口 / 个人时间窗口
```

## 输入预处理（兼容旧版口径）
- 让用户提供：项目愿景 / 目标客户 / 已有资源 / 替代选项 / 储蓄跑道 / 时间窗口约束
- 若用户没说替代选项，主持人主动询问"如果不做这个，时间会用来做什么"——立项决策必须看机会成本

## 输出额外要求
- **三种情景下的 5 年回报区间**（best/expected/worst）
- **机会成本对照表**：列出至少 2 个替代用途及其预期回报
- **3 道止损闸**：W4 / W8 / W12（或 M1 / M3 / M6）的硬止损条件
- **退出路径**：成功后退出 / 失败后退出 各一条

## 已知陷阱
- **创始人偏见放大**：用户对自己的项目天然乐观，必须强配怀疑派和红队
- **沉没成本陷阱**：如果用户"已经在做了"，注意区分"继续 vs 暂停"和"重新评估"
- **机会成本盲区**：用户经常忘记"做 A 意味着不做 B"，必须主动追问

## 与其他 skill 联动

- 决策结果若 = 暂缓，建议用户写"再启动条件"清单（哪些信号出现时回来重跑本 skill）
