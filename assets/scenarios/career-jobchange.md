# 场景预设：职业 / 跳槽决策

## 触发关键词
"换工作" / "要不要跳槽" / "这个 offer 接吗" / "升职加薪还是裸辞" / "要不要转行"

## 推荐角色配置（5 角色 standard mode）
1. Bayesian 更新派（同类决策成功率 base rate）
2. 投资人（机会成本 + 长期收益）
3. 战略派（5 年职业格局）
4. 怀疑派（offer 背后的隐性问题）
5. Steelman 派（构造"留下来"和"走"两边的最强版本）

## 推荐 mode
`standard`（3 轮 + 红队） — 职业决策不可逆性中等偏高

## 议题重述格式
`【议题】是否接受 [新机会]？现有 [现职] 是否值得放弃？`
`【类型】判断型 / 选择型`
`【成本】高（不可逆 + 跨年级影响）`

## required_inputs（v0.3.0 强制门禁 — 缺一即拒启动辩论）

```yaml
required_inputs:
  - 现职薪资包：base / bonus / equity 年化（不能只说"薪资"，必须三件套）
  - 新 offer 薪资包：base / bonus / sign-on / equity（同上三件套 + sign-on）
  - 期权细节：数量 / strike price / 当前估值 / vesting schedule（含 cliff）/ post-termination exercise window
  - scope 对照：团队规模 / 汇报对象 / 负责模块 / 决策权重（4 元素，不能只说"做什么"）
  - 决策窗口：offer deadline / 个人决策时长承诺
  - 当下生命阶段：房贷 / 家庭 / 现金流缓冲 / 可承担风险时长
```

主持人在 R1 启动前对照检查；缺关键字段 → 按 `assets/mechanisms/abstain-protocol.md` §5 输出索数据模板，**不启动辩论**。

## 输入预处理（兼容旧版口径）
让用户给出：
- 现职 vs 新机会的 7 维对照（薪资 / 成长 / 风险 / 距离家 / 团队 / 行业 / 老板）
- 决策窗口（offer deadline）
- 个人当下阶段：是否养家 / 还房贷 / 攒钱跑路 / 单身可冒险

## 输出额外要求
- **7 维对照打分表**
- **5 年终局推演**（best/expected/worst 三个剧本）
- **隐性条件挖掘**：股权 vesting / 竞业协议 / 试用期淘汰率
- **"留下"vs"走"双方最强版本**

## 已知陷阱
- **草坪综合征**：邻居家草坪更绿——必须客观评估当前职位被低估的部分
- **新鲜感偏好**：人们对"未到手的"高估、对"已到手的"低估
- **沉没成本陷阱**：在现公司"已经投入 N 年"——不应作为留下理由
