# 机制：诚实进度指标（Progress Markers）

> **目的**：让用户**真实感受到辩论的进展**，而不是用"激烈交锋""深度讨论"等空洞措辞营造伪深度。
>
> **核心原则**：所有指标必须由**真实计算**得出，不允许 agent 估算或拍脑袋。

## 6 项核心指标

### 1. 轮次（Rounds）
- **定义**：实际跑完的辩论轮数
- **格式**：`3/3` 或 `2/3 (R3 因共识达成提前结束)`
- **说明**：少于预设轮数必须解释原因

### 2. 发言数（Statements）
- **定义**：总发言条数（包括补充质询）
- **格式**：`18 条 (R1: 6 / R2: 6 / R3: 6)`
- **说明**：分轮统计，让用户看到节奏

### 3. 立场改变次数（Mind Changes）
- **定义**：在 mind-change-table.md 标准下被记录为"立场移动"的次数
- **格式**：`2 次（数据派 R1→R2 从 +0.7 → +0.3；战略派 R2→R3 从 -0.5 → +0.2）`
- **说明**：
  - **0 次**是危险信号——可能是 premature convergence 或大家从一开始就同意，主持人应在裁决卡标注"本场无立场改变，需警惕共识虚假"
  - **>5 次**也是危险信号——可能是 sycophancy 互相迎合

### 4. 未解分歧（Unresolved Disagreements）
- **定义**：辩论结束时仍在分歧矩阵中保留的论点数
- **格式**：`1 项未解（关于 [具体论点]，多空双方皆有依据）`
- **说明**：
  - **0 项未解**通常是过早收敛——主持人应主动列出"虽然达成共识但仍有不确定性的方向"
  - 未解分歧本身就是辩论的诚实输出，不要为了"漂亮的结论"强行收敛

### 5. 引证密度（Citation Density）
- **定义**：包含具体论据/数据/案例引用的发言占比
- **格式**：`87% (18 条中有 16 条带具体引证)`
- **计算方式**：
  - 包含数字 / 公式 / 具体场景描述 / 具体案例 / 具体源 → 计为有引证
  - 仅有"我认为""通常来说""一般而言""可能"等模糊措辞 → 不计为有引证
- **说明**：
  - 低于 60% 警示用户辩论质量偏低
  - 100% 也可能是问题——可能 agent 在堆砌看似具体的伪数据

### 6. 置信度校准（Confidence Calibration）
- **定义**：最终结论的加权置信度
- **格式**：`60% (基于支持者×1.0 - 反对者×0.5 - 红队折扣 12%)`
- **说明**：
  - 必须显式给出计算公式
  - 不允许只输出"较高置信度"等模糊表述
  - 30-50% 的置信度是常态而非缺陷——"敢于不确定"本身是质量

## 输出格式（裁决卡顶部固定挂）

```markdown
## 诚实进度行

| 指标 | 值 | 信号 |
|------|-----|------|
| 轮次 | 3/3 | ✅ 完整 |
| 发言数 | 18 (R1:6 / R2:6 / R3:6) | ✅ 节奏均衡 |
| 立场改变 | 2 次 | ✅ 有真实交锋 |
| 未解分歧 | 1 项 | ⚠️ 已显式标注 |
| 引证密度 | 87% | ✅ 高质量 |
| 置信度 | 60% | ⚠️ 中等——重大不可逆决策建议补强 |
```

## 危险信号自动告警

主持人在生成裁决卡时若检测到以下情况必须**显式标注**：

- 立场改变 = 0 → 警告"无立场改变——可能是 premature convergence 或议题本身共识度高"
- 立场改变 > 角色数 × 1.5 → 警告"立场频繁变动——可能是 sycophancy"
- 引证密度 < 60% → 警告"引证密度偏低——结论不应作为重要决策依据"
- 红队折扣 > 25% → 警告"红队大幅压低置信度——共识未通过压测"
- 未解分歧 ≥ 3 → 警告"未解分歧过多——建议二次辩论或补充信息后重审"

## 防剧场化硬约束

- ❌ 禁止用"激烈/深度/精彩/全面"等形容词替代真实指标
- ❌ 禁止伪造数据（如"经过 1000 轮深度推演"——本 skill 实际只跑 3 轮）
- ✅ 真实数字本身就有说服力——`87%` 比 `非常高的引证密度` 更可信
- ✅ 不好看的指标（如立场改变 0 / 未解 3）必须如实显示，不要美化
