# 机制：立场变迁表（Mind Change Table）

> **目的**：表格化展示**每个角色在跨轮次中的立场变化**，让用户看到"辩论真的在改变想法"，而不是各说各话。
>
> **底层洞察**：一个好辩论的标志不是"达成完美共识"，而是"参与者愿意基于新证据移动立场"。立场变化 = 论据真正起了作用。

## 立场量化方法

每位角色在**每一轮结束**时被主持人评估当前对**核心议题主张**的立场，使用 **-1.0 到 +1.0 的连续量表**：

| 取值 | 含义 |
|------|------|
| +1.0 | 完全支持（"无条件推进"） |
| +0.5 | 倾向支持（"附条件推进"） |
| 0.0 | 中立 / 待定 |
| -0.5 | 倾向反对（"重大风险，建议暂缓"） |
| -1.0 | 完全反对（"必须否决"） |

**评估方式**：主持人**根据角色发言原文**判断（不是问 agent 自报），关键词锚定：
- "强烈推荐 / 必须做"→ +1.0
- "建议推进 / 看好"→ +0.5
- "需要更多信息 / 各有利弊"→ 0.0
- "风险大 / 建议谨慎 / 暂缓"→ -0.5
- "反对 / 不建议 / 风险不可控"→ -1.0

## 表格格式

```markdown
## 立场变迁表

| 角色 | R1 立场 | R2 立场 | R3 立场 | 净变化 | 关键转折点 |
|------|---------|---------|---------|--------|-----------|
| 数据派 | +0.7 | +0.3 | +0.3 | -0.4 | R1→R2 因用户派指出留存数据样本偏差 |
| 用户派 | -0.3 | -0.3 | -0.5 | -0.2 | R2→R3 因看到迁移成本计算 |
| 怀疑派 | -0.7 | -0.5 | -0.3 | +0.4 | R1→R2 因执行派给出降级方案 |
| 执行派 | +0.5 | +0.5 | +0.7 | +0.2 | R2→R3 因获得人力补充承诺 |
| 投资人 | 0.0 | +0.3 | +0.5 | +0.5 | R1→R3 ROI 数字逐步明朗 |

**整体趋势**：从初始平均 +0.04 → 最终 +0.14（轻度向支持移动 +0.10）
**最大变化角色**：投资人（+0.5）和怀疑派（+0.4）
**立场最稳定**：用户派（仅 -0.2）
```

## 关键转折点（Pivot Points）记录规则

每个出现 ≥ 0.3 立场移动的格子必须在"关键转折点"列填入：
- **由谁的发言触发**（哪个角色的哪轮）
- **核心论据是什么**（一句话）

格式：`R[X]→R[Y] 因 [角色] 提到的 [核心论据]`

## 净变化的解读

| 净变化绝对值 | 含义 | 信号 |
|-------------|------|------|
| 0 | 立场完全没动 | ⚠️ 警示——可能根本没听对方 |
| 0.1-0.3 | 轻微调整 | ✅ 正常——边际更新 |
| 0.4-0.7 | 显著移动 | ✅ 高质量——真听进去了 |
| 0.8-2.0 | 立场翻转 | ⚠️ 警示——可能是 sycophancy 或被压力扭转 |

**整体平均净变化绝对值**：
- < 0.1 → 辩论无效（共识虚假或各说各话）
- 0.1-0.3 → 健康
- 0.3-0.5 → 高质量辩论
- > 0.5 → 警惕从众

## 多维度立场（可选高级用法）

对**复杂议题**（如多个子选项），可拆为多个子立场维度：

```markdown
| 角色 | 子议题A立场 | 子议题B立场 | 子议题C立场 |
|------|-----------|-----------|-----------|
| 数据派 | +0.7→+0.3 | +0.5→+0.5 | -0.3→-0.5 |
```

仅在 `deep` mode 且议题包含 ≥ 3 个独立子选项时启用。

## 防伪规则

1. **立场必须从原文证据推导**——主持人在表格下方附 1-2 行该角色发言的关键句作为定档依据
2. **不允许出现"立场无变化但发言内容大幅扩展"** —— 那意味着发言只是在堆字
3. **关键转折点不能虚构**——必须能从对话原文找到对应触发句

## 与其他机制的协同

- 立场变迁表 → 喂给 progress-markers.md 中的"立场改变次数"
- 立场移动 ≥ 0.3 的发言 → 自动成为 highlight-extraction.md 的"转折句"候选
- 净变化为 0 的角色多 → 在 PITFALLS 中触发 "P14: 立场冰冻"告警
