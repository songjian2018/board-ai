# Dry-Run 测试用例

> 这些用例覆盖了 4 大场景预设。每次 skill 重大更新都应跑这些用例。

---

## 用例 1：项目立项决策（核心使用场景）

### 输入
> 我现在在做 AI 投放工具的内部 skill 跑通，6 月底之前要决定是否在 Q3 把它升级为对外 SaaS 产品。我有 12 周时间，3-5 万启动资金。同期我还想做 FitMind 健身 App、雀圣牌游戏、面试求职 P7。

### 期望流程
1. **议题诊断**：议题=判断型，决策成本=高（不可逆 12 周投入）
2. **场景识别**：匹配 `project-go-no-go.md`，启用 deep mode
3. **角色召集**：7 角色（战略+投资+用户+数据+执行+怀疑+未羊）+ 红队
4. **第一轮盲评**：每角色独立给立场+论据+置信度
5. **第二轮交叉质询**：识别共识/分歧/事实矛盾
6. **第三轮**：再次质询+共识收敛
7. **红队压测**：找未声明假设 TOP 3 + 失败情景
8. **裁决**：输出含三条路径（最优/次优/兜底）+ 关键不确定性的决策卡

### 期望输出关键特征
- 三条推荐路径之间存在真实取舍而非线性退化
- 红队提出至少 1 个未声明的脆弱假设（如巨量 Scope 审批时长）
- 给出具体止损线（W4/W8/W12 的硬条件）
- 引用机会成本（求职 P7 / FitMind / 雀圣牌）
- 主持人裁决质量自检全部通过

### 失败征兆
- 所有角色都说"应该做"或"不应该做"——可能 Sycophancy
- 没有提到机会成本——创始人偏见放大
- 红队没找到任何脆弱假设——红队失职

## 用例 2：PRD 审查

### 输入
> 帮我审一下 growth-agent 的 v2.0 PRD，特别看看预处理层和工作流之间的衔接是不是合理。

### 期望流程
1. 议题诊断：审查型 + 中等成本
2. 匹配 `prd-review.md`，5 角色 standard mode
3. PRD 结构化提炼（目标/指标/场景/模块/依赖/风险）
4. 三轮辩论 + 红队
5. 输出含**冲突候选清单** + **指标可观测性评估** + **采纳风险分级**

### 期望输出关键特征
- 数据派指出指标定义口径问题
- 用户派指出采纳门槛和 AHA 时机
- 未羊视角对比业务规则手册（E-007 / B-008）找口径不一致
- 至少 3 条具体改进建议（非"建议优化"这种水词）

## 用例 3：跨文档冲突审计（fast mode）

### 输入
> 我有 3 份文档：业务规则手册 v1.2 / growth-agent PRD v2.0 / 决策矩阵 B-008。它们对"质量分阈值"的口径好像不一致，帮我对一下。

### 期望流程
1. 议题诊断：审查型，决策成本=低（可逆）
2. 匹配 `cross-doc-audit.md`，4 角色 fast mode
3. 文档预扫识别冲突候选项
4. 用户确认候选清单
5. 两轮辩论（跳过红队）
6. 输出**冲突清单表**，每条含 before/after 取值建议 + 理由 + 涉及改动点

### 期望输出关键特征
- 不直接修改文档（用户偏好"先列冲突让我拍板"）
- 以业务规则手册为 single source of truth
- 阈值口径偏好"连续无空档 + 含出价档"
- 改动建议可一条条点选

## 用例 4：多方案取舍（资源争夺）

### 输入
> 我手上 4 个项目：AI 投放 SaaS / FitMind / 雀圣牌 / 求职 P7。储蓄能撑 6 个月。该选哪个，可以并行吗？

### 期望流程
1. 议题诊断：选择型 + 高决策成本
2. 匹配 `option-tradeoff.md`，5 角色 standard mode
3. 主持人识别 4 个选项的资源争夺关系（时间/钱/认知带宽）
4. 每个角色独立评估每个选项，不只是"哪个最好"
5. 输出**方案对比矩阵** + **决策树** + **节奏建议**

### 期望输出关键特征
- 矩阵中至少有 1 个维度（如时间）显示选项间互斥
- 决策树覆盖 ≥ 3 种前提下的不同选择
- 节奏建议给出"分阶段串行 vs 强行并行"的具体取舍

## 自动化验证 checklist

每次跑完上述任一用例后，对照下列项目打勾：

- [ ] 议题诊断输出明确（类型 + 成本 + 重述）
- [ ] 角色配置符合场景预设
- [ ] 第一轮发言内部独立性肉眼可见（不互引）
- [ ] 每个发言都带 `[confidence: XX%, type]`
- [ ] 至少有 1 个角色立场和其他人不同（多样性证明）
- [ ] 红队（如启用）至少提出 1 个具体未声明假设
- [ ] 主持人裁决卡 5 大模块齐全（结论/共识/分歧/不确定性/路径）
- [ ] 质量自检 4 项全部通过 OR 已挂警告
- [ ] Token 消耗 < 30k（standard mode）/ < 60k（deep mode）
