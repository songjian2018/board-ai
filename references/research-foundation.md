# 学术研究依据汇总

本 skill 的设计**严格基于** 2025-2026 年多 agent 辩论的学术研究成果。每个核心机制都能追溯到具体论文/产品。

## 核心论文清单

### 1. Sycophancy / 趋同问题
- **arXiv 2509.05396**：Understanding Failure Modes in Multi-Agent Debate
- 核心发现：LLM 在多 agent 辩论中倾向于附和他人，从正确转向错误
- 我们的应用：P1 规避手段 + 角色池"赞同必带反对点"硬约束

### 2. Identity Bias / 身份偏差
- **arXiv 2510.07517**：Measuring and Mitigating Identity Bias in Multi-Agent Debate
- 核心发现：agent 因身份标签产生偏好；Response Anonymization 是有效解
- 我们的应用：第一轮盲评机制（Round 1 Anonymous）

### 3. 是否真的在辩论 / 辩论质量评估
- **arXiv 2511.07784**：Can LLM Agents Really Debate? A Controlled Study
- 核心发现：辩论常沦为"多数投票"；有效辩论需多数压力抑制独立修正、推理基于逻辑非修辞
- 评估标准：纠正率 + 理性行为
- 我们的应用：置信度加权裁决（不是纯投票）+ 主持人识别"水词发言"

### 4. 思维定势突破
- **DMAD (Diverse Multi-Agent Debate)**：openreview t6QHYUOQL7
- 核心发现：异构 reasoning style 突破"mental set fixation"
- 我们的应用：角色池 reasoning style 强制 ≥ 4 种异构

### 5. 投票还是辩论
- **Debate or Vote** (Choi & Zhu)：semanticscholar
- 核心发现：劣质辩论 < 简单投票；优质辩论 > 投票
- 我们的应用：置信度加权裁决（同时具备辩论的深度和投票的鲁棒性）

### 6. 共识效率
- **Towards Efficient and Effective Consensus in Multi-Agent LLM** (ACL 2025)
- 核心发现：达成共识的成本和质量需平衡
- 我们的应用：4 阶段流程 + token 预算约束（PITFALLS P10）

## 核心产品对照

### 1. Crucible (RoundTable Labs)
- **优势**：成熟产品 + $19/月 SaaS + 开源自托管版
- **缺陷**：闭源核心 + 角色相对固定
- **我们的吸收**：4 阶段流程 + 红队最后压测
- **我们的差异化**：角色完全可定制 + 内嵌中文场景库（PRD/立项/跨文档）

### 2. MAGI / 产品大师（开源中文）
- **优势**：强制工作流 + 检查点机制 + Skill 校验
- **缺陷**：只 3 角色 + 写死 PRD 场景
- **我们的吸收**：强制工作流 + 检查点
- **我们的差异化**：通用 6+1 角色池 + 多场景预设

### 3. 团子 (Nextie / 小冰李笛)
- **优势**：异构智能体议会 + Token 效率 + 同步思考可见
- **缺陷**：内测产品 + 不可定制 + 依赖网页
- **我们的吸收**：异构 reasoning style + 中间过程 TodoList 可视化
- **我们的差异化**：本地 skill + 完全可控 + 可串联其他 skill

## 我们的独特贡献

1. **场景化预设**：11 个场景（4 工作 + 7 通用决策：职业/财务/技术/人际/健康/伦理/兜底）—— 业内首次覆盖通用人生决策
2. **角色专业感锚点**（v0.2 新增）：每个角色卡的 `thought_lineage`（思维谱系）+ `will_not_do`（专业边界反向锚定）+ `signal_strength_calibration`（自评等级）—— 业内首次将角色"专业感"算法化
3. **反幻觉链路**：来源标注 + 置信度声明 + 加权算法显式化 —— 学自 growth-agent 防幻觉方案
4. **决策成本驱动 mode**：高成本议题强制 deep / 低成本议题强制 fast —— 资源最优
5. **质量自检 + 警告机制**：跑完后自动判断辩论质量，不达标挂明显警告 —— 业内罕见
6. **反剧场化机制**（v0.2 新增）：300 字硬上限 + 禁要求 agent 输出金句 + 进度指标真实计算三条硬约束 —— 解决"伪深度"行业通病
7. **金句客观提取算法**（v0.2 新增）：4 标准（碰撞/转折/盲区/凝练）+ 一票否决无证据金句 —— 替代主观"挑金句"
8. **立场变迁量化**（v0.2 新增）：±1.0 连续量表 + 关键转折点必须可追溯到原文 —— 让辩论价值"看得见"
9. **诚实进度指标**（v0.2 新增）：6 项真实计算指标 + 5 种危险信号自动告警 —— 替代"激烈/深度/精彩"等空洞形容词

## v0.2 设计理念论据：信号化 vs 剧场化

### 核心命题
"让用户感受到辩论的价值"有两条路径：
- **剧场化**（劣解）：增加输出量、堆砌看似深刻的金句、用形容词渲染深度——本质上是用 token 灌水换感知
- **信号化**（v0.2 选择）：不增加输出量，**重新组织信号优先级**——把真实发生的"立场移动 / 真证据 / 真分歧 / 真转折"突出展示

### 支持论据

**1. OpenAI o1 思考过程展示的价值前提**
- o1 把"思考过程"显示给用户广受好评，但价值前提是**思考真实发生**
- 表演性的"思考过程"反而加深认知负担、误导用户
- 类比：本 skill 的"立场变迁表"+"金句精选区"也只在**真实变化/真实凝练**发生时才有价值

**2. Tetlock 超级预测者的"诚实校准"**
- 《超级预测》研究发现：高质量预测的标志不是"高自信 + 漂亮叙事"，而是**校准的不确定度 + 频繁更新**
- 本 skill 的"诚实进度指标"（含 30-50% 置信度被视为常态而非缺陷）即是 Tetlock 原则的工程化

**3. Tufte 信息密度原则**
- Edward Tufte《The Visual Display of Quantitative Information》提出"data-ink ratio"——信息载体应让数据本身占主导
- 本 skill 的"诚实进度行"用真实数字表（87% / 2 次 / 60%）替代修饰词，符合 Tufte 原则

**4. 反对意见的考虑**
- 反对意见 1："不堆金句，用户感知不到深度" → 反驳：堆金句的是 ChatGPT 默认风格，用户已审美疲劳；真实数据 + 真实转折反而稀缺
- 反对意见 2："300 字上限会限制深度" → 反驳：研究显示有效论点的最佳长度 100-250 字（参考 SSC blog post 阅读保留率研究），超过反而稀释
- 反对意见 3："禁止 agent 输出金句太严" → 反驳：保留主持人提取金句的通道，且主持人按客观标准提取——结果反而更高质量

### 取舍记录

我们**接受**的代价：
- 单次输出比剧场化产品视觉冲击力低
- 用户初次使用可能感觉"不够 wow"

我们**拒绝**的代价：
- 用户被伪深度误导而做错决策
- 长期信任被"看起来很专业但不可验证"消耗
- 偏离"真正改进决策质量"这一核心目的
